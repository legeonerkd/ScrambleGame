# История изменений ScrabbleGame

## [2.0.0] - 2024-04-06

### Добавлено ✨
- **Новое правило**: Минимальная длина слова - 3 буквы
  - Все слова (основные и побочные) должны содержать минимум 3 буквы
  - Добавлена проверка в `GameState.can_place()` (models/game_state.py:199)
  
- **Новое правило**: Защита составленных слов
  - Нельзя ставить буквы на уже занятые клетки
  - Добавлена проверка в `GameState.can_place()` (models/game_state.py:180)
  
- **Модульная архитектура**:
  - `models/dictionary.py` - класс ScrabbleDictionary с документацией
  - `models/player.py` - класс Player (для будущего использования)
  - `models/game_state.py` - класс GameState с типами и документацией
  
- **Тестирование**: 43 модульных теста
  - `tests/test_game_state.py` - 9 тестов инициализации и управления
  - `tests/test_move_validation.py` - 11 тестов валидации ходов
  - `tests/test_scoring.py` - 13 тестов подсчета очков
  - `tests/test_new_rules.py` - 10 тестов новых правил
  - Покрытие игровой логики ~80%

- **Документация**:
  - `REFACTORING.md` - история рефакторинга
  - `RULES.md` - подробные правила игры с примерами
  - `CHANGELOG.md` - этот файл
  - Обновлен `README.md` с новой информацией

- **Инструменты разработки**:
  - `pytest.ini` - конфигурация тестов
  - `requirements.txt` - зависимости проекта
  - `clean_dictionary.py` - скрипт очистки словаря

### Изменено 🔄
- **Словарь**: Удалены все слова из 1-2 букв
  - Было: 34,038 слов
  - Удалено: 266 слов (сохранены в `removed_short_words.txt`)
  - Стало: 33,772 слова (минимум 3 буквы)
  
- **GameState**: Улучшена читаемость и документация
  - Добавлены типы аннотации для всех методов
  - Добавлены docstrings для всех классов и методов
  - Добавлен метод `_next_player()` вместо дублирования кода
  - Улучшен метод `_collect_word()` для корректной обработки None

- **engine.py**: Теперь импортирует из models для обратной совместимости
  
- **main.py**: Использует новые модули
  - Импортирует `ScrabbleDictionary` из `models.dictionary`
  - Импортирует `GameState` из `models.game_state`

### Исправлено 🐛
- Исправлена проблема с типами в `_collect_word()` при обработке `None`
- Улучшена обработка ошибок при валидации ходов
- Добавлены более информативные сообщения об ошибках

### Технические детали 🔧
- Python 3.6+ (рекомендуется 3.12+)
- Все тесты проходят: `pytest tests/ -v` ✅
- Обратная совместимость сохранена через `engine.py`
