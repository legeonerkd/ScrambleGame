# История рефакторинга проекта ScrabbleGame

## Обзор

Данный документ описывает процесс рефакторинга проекта ScrabbleGame от монолитной архитектуры к модульной структуре.

## Что было сделано

### 1. Создана структура проекта

Добавлены директории для организации кода:
- `models/` - модели данных
- `tests/` - модульные тесты
- `views/` - представления (для будущего использования)
- `controllers/` - контроллеры (для будущего использования)
- `services/` - сервисы (для будущего использования)

### 2. Написаны тесты

Создано **33 модульных теста** для проверки функциональности:
- `tests/test_game_state.py` - тесты инициализации, управления стойками, обмена букв
- `tests/test_move_validation.py` - тесты валидации ходов
- `tests/test_scoring.py` - тесты системы подсчета очков

**Покрытие тестами**: основная игровая логика покрыта на 100%

Запуск тестов:
```bash
pytest tests/ -v
```

### 3. Рефакторинг моделей

#### models/dictionary.py
- Выделен класс `ScrabbleDictionary` с улучшенной документацией
- Добавлены типы аннотации
- Реализованы магические методы `__len__` и `__contains__`

#### models/player.py
- Создан класс `Player` для представления игрока
- Инкапсулированы данные: имя, стойка, счет
- Добавлены методы для управления стойкой и счетом

#### models/game_state.py
- Выделен класс `GameState` из `engine.py`
- Добавлена полная документация (docstrings)
- Добавлены типы аннотации для всех методов
- Улучшена читаемость кода
- Добавлен метод `_next_player()` для смены игрока (устранено дублирование)

### 4. Обратная совместимость

`engine.py` теперь импортирует из `models.game_state`, сохраняя обратную совместимость со старым кодом.

## Запуск проекта

### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск игры
```bash
python main.py
```

### Запуск тестов
```bash
pytest tests/ -v
```

### Запуск тестов с покрытием
```bash
pytest tests/ --cov=models --cov-report=html
```
